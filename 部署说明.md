## 方案一：Netlify 部署（推荐 - 最简单）

### 步骤：

1. **打开网站**: 浏览器访问 [https://app.netlify.com](https://app.netlify.com)

2. **注册/登录**: 可以用 GitHub 账号直接登录

3. **拖拽部署**:
   - 登录后点击 "Add new site" → "Deploy manually"
   - **直接把下面这个文件夹拖进去**：
   
   ```
   D:\桌面\所有作业\实验报告\web\work.4\book-front\dist\
   ```

4. **完成!** 几秒后你会得到一个类似这样的链接：
   ```
   https://xxxxx.netlify.app
   ```

5. **手机电脑都能打开**，复制链接即可访问！

---

## 方案二：GitHub Pages 部署

1. 在 GitHub 创建新仓库，把 `book-front/dist/` 里的文件上传
2. 进入仓库 Settings → Pages → 选择 main 分支 /public
3. 等待部署完成后访问 `https://用户名.github.io/仓库名/`

---

## 重要说明

### 关于后端 API

你的系统需要后端支持（登录、图书增删改查），当前前端连接的是 `localhost:3000` 的后端服务。

**两种解决方式：**

#### 方式 A：仅展示前端效果
- 上面的部署方式可以让前端页面正常打开
- 登录和图书功能会提示"请求失败"（因为后端没部署）
- 适合给老师展示界面效果

#### 方式 B：前后端都部署（完整功能）

**后端部署到 Railway.app（免费）：**

1. 打开 [https://railway.app](https://railway.app) 注册
2. 创建新项目 → Deploy from GitHub repo
3. 需要先将整个 `work.4` 目录推送到 GitHub
4. 添加 MySQL 数据库插件（Railway 免费提供）
5. 设置环境变量指向数据库

**或者使用 SQLite 替代 MySQL（更简单）：**

如果你愿意把后端改为使用 SQLite（无需安装 MySQL），我可以帮你改造，这样只需要一个免费平台就能前后端一起部署。
